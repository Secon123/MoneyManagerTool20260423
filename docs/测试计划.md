## 3. 测试策略

### 3.1 接口测试
- 使用 Postman 对每个接口进行正向和异常测试
- 验证 Token 鉴权机制
- 验证数据正确写入数据库

### 3.2 前端功能测试
- 模拟真实用户操作
- 验证页面交互和数据一致性
- 验证响应式布局

### 3.3 回归测试
- 修复 Bug 后重新验证
- 确保修复不影响其他功能

## 7. 风险与应对

| 风险 | 应对措施 |
|------|----------|
| 后端接口不稳定 | 提前与后端同学确认接口状态 |

| 时间不足 | 优先保证核心功能（P0）测试通过 |
