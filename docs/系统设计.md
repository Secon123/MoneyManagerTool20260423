# 系统设计

**项目名称**：个人记账管理系统  
**更新日期**：2026年5月18日  

---

## 1. 系统架构

本系统采用 **B/S 架构**，前后端分离模式。

### 1.1 架构图

![系统架构图](./report/assets/fig_architecture.png)


### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue.js + ECharts | 页面渲染 + 图表展示 |
| 后端 | Spring Boot | RESTful API 服务 |
| 数据库 | MySQL | 数据持久化存储 |
| 认证 | JWT | 用户身份认证 |
| 版本控制 | Git + GitHub | 代码托管与协作 |

## 2. 模块设计

### 2.1 模块划分

| 模块 | 职责 | 主要功能 |
|------|------|----------|
| 用户模块 | 用户身份认证与管理 | 注册、登录、Token鉴权 |
| 记账模块 | 收支记录管理 | 添加、编辑、删除、查询收支记录 |
| 统计模块 | 数据可视化分析 | 月度报表、支出饼图、收支趋势折线图 |
| 预算模块 | 预算控制与提醒 | 设置分类预算、月度限额、超额提醒 |

### 3.2 主要数据表

**表1：user（用户表）**

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(255) | NOT NULL | 密码（bcrypt加密） |
| email | VARCHAR(100) | UNIQUE | 邮箱 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

**表2：transaction（收支记录表）**

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 记录ID |
| user_id | INT | NOT NULL, FOREIGN KEY → user(id) | 所属用户 |
| type | ENUM('income','expense') | NOT NULL | 类型：收入/支出 |
| amount | DECIMAL(10,2) | NOT NULL | 金额 |
| category | VARCHAR(50) | NOT NULL | 分类（餐饮/交通/购物等） |
| date | DATE | NOT NULL | 日期 |
| note | TEXT | | 备注 |

**表3：budget（预算表）**

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 预算ID |
| user_id | INT | NOT NULL, FOREIGN KEY → user(id) | 所属用户 |
| category | VARCHAR(50) | NOT NULL | 分类名称 |
| monthly_limit | DECIMAL(10,2) | NOT NULL | 月度限额 |
| month | DATE | NOT NULL | 预算月份 |



## 4. 接口设计

### 4.1 用户模块

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/register` | POST | 用户注册 |
| `/api/auth/login` | POST | 用户登录 |

### 4.2 记账模块

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/transactions` | POST | 添加收支记录 |
| `/api/transactions/{id}` | PUT | 修改收支记录 |
| `/api/transactions/{id}` | DELETE | 删除收支记录 |
| `/api/transactions` | GET | 查询收支列表 |

### 4.3 统计模块

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/statistics/monthly-report` | GET | 月度报表 |
| `/api/statistics/expense-pie` | GET | 支出饼图数据 |
| `/api/statistics/trend` | GET | 收支趋势折线图 |

### 4.4 预算模块

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/budgets` | POST | 设置预算 |
| `/api/budgets` | GET | 查询预算 |

### 4.5 通用响应格式

```json
{
    "code": 200,
    "message": "success",
    "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，200表示成功 |
| message |	string | 响应消息 |
| data | object | 响应数据 |
