# 需求分析

**项目名称**：个人记账管理系统  
**更新日期**：2026年5月18日  

---

## 1. 项目背景

随着移动支付和线上消费的普及，个人日常资金流动日益频繁、渠道分散。大学生及年轻群体普遍存在“钱花哪儿了”不清楚、月底对账困难、缺乏预算意识的现实问题。本系统旨在通过一个轻量级、可落地的个人记账管理系统，帮助用户培养良好的财务记录与管理习惯。

---

## 2. 目标用户与使用场景

### 2.1 目标用户
- 在校大学生
- 刚参加工作的年轻人
- 注重日常开销记录的个人用户

### 2.2 使用场景
| 场景 | 描述 |
|------|------|
| 日常收支记录 | 用户可随时记录餐饮、交通、购物、工资等收入和支出 |
| 月度财务回顾 | 按月查看收支报表，了解资金流向 |
| 分类预算控制 | 为餐饮、娱乐等分类设置月度预算，防止超支 |
| 个人资金流水查询 | 支持按时间、分类检索历史记录 |

---

## 3. 功能需求

### 3.1 功能清单

| 功能模块 | 功能描述 |
|----------|----------|
| 用户注册 | 新用户填写用户名、密码、邮箱进行注册 |
| 用户登录 | 已注册用户通过用户名和密码登录系统 |
| 添加收入/支出 | 记录金额、分类、日期、备注等信息 |
| 编辑收支记录 | 修改已有记录的金额、分类或备注 |
| 删除收支记录 | 删除不需要的流水记录 |
| 收支列表查询 | 按时间范围查看收支流水，支持分页 |
| 分类统计（饼图） | 按支出分类展示占比，辅助分析消费结构 |
| 月度报表 | 按月统计总收入、总支出及结余 |
| 预算设置 | 为每个支出分类设置月度预算限额 |
| 超额提醒 | 当某分类支出超过预算时提示用户 |

### 3.2 业务流程

用户注册 → 登录系统 → 添加收支记录 → 查看列表和统计图表 → 按需设置预算 → 月底回顾月度报表



---

## 4. 非功能需求

### 4.1 性能要求
- 页面加载时间 ≤ 2 秒
- 支持并发访问用户数 ≥ 20 人
- 接口响应时间 ≤ 500 毫秒（正常负载下）

### 4.2 安全要求
- 用户密码采用 bcrypt 等算法加密存储，不保存明文
- 接口使用 JWT Token 进行身份认证，防止未授权访问
- 对用户输入进行校验和过滤，防止 SQL 注入
- 用户数据相互隔离，只能查看自己的收支记录

### 4.3 兼容性要求
- 支持 Chrome、Edge 等主流浏览器最新版本
- 前端采用响应式布局，适配手机端（宽度 ≤ 768px）
- 后端接口遵循 RESTful 风格，返回统一 JSON 格式

---

## 5. 需求优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0（必须有） | 用户注册、登录、添加收支、收支列表 | 核心记账功能 |
| P1（重要） | 编辑记录、删除记录、月度报表 | 完整记账流程 |
| P2（较好有） | 支出饼图、预算设置、超额提醒 | 可视化与预算控制 |
| P3（可选） | 多账本、数据导出、趋势预测 | 扩展功能 |
