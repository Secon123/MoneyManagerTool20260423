# BUG-006：前端无法登录（前后端未连接 / 未部署）

**发现人**：曾昌缙  
**发现时间**：2026-05-05  
**严重程度**：🔴 阻塞（Blocking）  
**状态**：待解决  

## 问题描述
在进行前端页面测试时，无法完成登录操作。  
根本原因并非前端代码错误，而是**前后端未建立实际连接**，且前端未部署到可访问的服务器环境。
当前前端同学提供的是**本地文件地址**（如 `C:\...\index.html`），而非 `http://` 开头的可访问网址，导致：
- 测试同学无法打开页面
- 页面无法调用后端接口
- 无法进行任何功能验证

## 复现步骤
1. 前端同学提供文件地址（非 http 地址）
2. 尝试在浏览器中打开
3. 页面无法正常调用后端接口
4. 登录功能不可用

## 预期结果
- 前端应通过 `http://` 地址访问（如 `http://localhost:5173` 或 `http://192.168.x.x:5173`）
- 前端代码应配置后端服务地址：`http://172.17.74.231:8080`
- 登录成功后，页面能正常跳转并保存 token

## 实际结果
- 前后端未连通
- 前端未部署到服务器或本地开发服务器
- 登录功能无法验证

## 阻塞范围
- 所有前端页面功能（注册、登录、记账、统计、预算等）均无法测试

## 解决建议（需前端同学完成）
1. 在本地启动前端开发服务器（如 `npm run dev -- --host`）
2. 将 **Network** 地址（如 `http://192.168.x.x:5173`）提供给测试同学
3. 确保前端代码中配置了正确的后端地址：`http://172.17.74.231:8080`
4. 确保登录、注册等接口调用时正确携带参数和 token
