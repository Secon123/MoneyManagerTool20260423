# 数据库表字段说明

**数据库名称**：expense_tracker  
**更新日期**：2026年5月18日  

---

## 一、用户表（user）

存储系统用户的基本信息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PRIMARY KEY, AUTO_INCREMENT | 用户唯一标识，自增主键 |
| username | varchar(255) | NOT NULL, UNIQUE | 用户名，不可重复 |
| password | varchar(255) | NOT NULL | 登录密码（bcrypt加密存储） |
| email | varchar(255) | DEFAULT NULL, UNIQUE | 邮箱地址，可选，不可重复 |
| register_time | datetime | NOT NULL | 注册时间 |

**索引说明**：
- `uk_username`：用户名唯一索引
- `uk_email`：邮箱唯一索引

---

## 二、收支记录表（transaction）

存储用户的每一笔收入或支出记录。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PRIMARY KEY, AUTO_INCREMENT | 记录唯一标识，自增主键 |
| user_id | bigint | NOT NULL, FOREIGN KEY | 所属用户ID，关联 user(id)，级联删除 |
| type | varchar(10) | NOT NULL | 类型：INCOME（收入）/ EXPENSE（支出） |
| amount | decimal(12,2) | NOT NULL | 金额，精确到小数点后两位 |
| category | varchar(255) | NOT NULL | 分类（如：餐饮、交通、购物、工资） |
| date | date | NOT NULL | 交易日期 |
| remark | varchar(255) | DEFAULT NULL | 备注信息，可选 |

**外键约束**：
- `fk_transaction_user`：`user_id` 引用 `user(id)`，删除用户时级联删除其收支记录

**索引说明**：
- `idx_user_date`：按用户+日期查询优化
- `idx_user_type`：按用户+类型查询优化

---

## 三、预算表（budget）

存储用户为各支出分类设置的月度预算。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PRIMARY KEY, AUTO_INCREMENT | 预算唯一标识，自增主键 |
| user_id | bigint | NOT NULL, FOREIGN KEY | 所属用户ID，关联 user(id)，级联删除 |
| category | varchar(255) | NOT NULL | 支出分类名称（如：餐饮、交通） |
| month_limit | decimal(12,2) | NOT NULL | 月度预算限额 |
| month | varchar(7) | NOT NULL | 预算月份，格式：YYYY-MM（如 2025-05） |

**唯一约束**：
- `uk_user_category_month`：同一用户、同一分类、同一月份只能有一条预算记录

**外键约束**：
- `fk_budget_user`：`user_id` 引用 `user(id)`，删除用户时级联删除其预算

**索引说明**：
- `idx_user_month`：按用户+月份查询优化
