## 一、项目概述  【中期必填】

### 1. 项目背景
个人在日常财务管理中常面临记录分散、分类混乱、收支不明的问题。传统Excel管理易出错且无法实时统计。本项目旨在开发一个轻量级的个人记账管理系统，帮助用户便捷记录日常收支、查看资金流向，提升个人财务透明度。

### 2. 系统目标
实现用户注册登录、收入/支出记录管理、分类统计、月度报表查看、预算提醒等核心功能，提供清晰的个人资金流动可视化展示。

### 3. 开发环境
- 开发工具：VS Code / IntelliJ IDEA
- 数据库：MySQL
- 运行环境：Windows，浏览器（Chrome/Edge）+ 本地服务器
- 后端：Java Spring Boot
- 前端：HTML/CSS/JavaScript + ECharts + Vue.js（可选）

---

## 二、需求分析  【中期必填】

### 1. 功能需求
- 用户注册与登录
- 添加收入/支出（金额、分类、日期、备注）
- 编辑/删除流水记录
- 按分类统计支出占比（如餐饮、交通、购物）
- 月度收支报表与余额计算
- 预算设置与超额提醒
- 可视化图表展示（饼图/折线图）

### 2. 非功能需求
- **性能要求**：页面加载 ≤ 2秒，支持并发 ≥ 20人
- **安全要求**：密码加密存储（如bcrypt），防止SQL注入
- **兼容性要求**：支持Chrome/Edge，响应式布局适配手机端

---

## 三、系统设计  【中期必填】

### 1. 系统架构
采用 B/S 架构，前后端分离模式。
- 前端：负责页面展示和用户交互
- 后端：提供 RESTful API 接口
- 数据库：MySQL 存储数据

> 建议补充架构图（后续放到 `assets/fig_architecture.png`）

### 2. 模块设计
| 模块 | 职责 |
|------|------|
| 用户模块 | 注册、登录、用户信息管理 |
| 记账模块 | 收支记录的增删改查 |
| 统计模块 | 月度报表、分类占比饼图、收支趋势折线图 |
| 预算模块 | 预算设置、超额提醒 |

### 3. 数据库设计
#### 主要数据表

**user（用户表）**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| username | VARCHAR(50) | 用户名，唯一 |
| password | VARCHAR(255) | 密码（加密存储） |
| email | VARCHAR(100) | 邮箱 |
| created_at | TIMESTAMP | 注册时间 |

**transaction（收支记录表）**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 用户ID，外键 |
| type | ENUM | 类型：income/expense |
| amount | DECIMAL(10,2) | 金额 |
| category | VARCHAR(50) | 分类 |
| date | DATE | 日期 |
| note | TEXT | 备注 |

**budget（预算表）**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 用户ID，外键 |
| category | VARCHAR(50) | 分类 |
| monthly_limit | DECIMAL(10,2) | 月度限额 |
| month | DATE | 月份 |
