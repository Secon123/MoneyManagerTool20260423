# 个人记账管理系统 - 用户手册
 
**更新日期**：2026年5月18日  
**适用系统**：Windows / macOS / Linux（浏览器访问）

---

## 一、安装部署说明

### 1. 环境要求

| 组件 | 版本要求 |
|------|----------|
| JDK | 11 或更高版本 |
| MySQL | 5.7 或 8.0 |
| Node.js | 16 或更高版本 |
| 浏览器 | Chrome / Edge 最新版 |

### 2. 后端部署步骤

#### 2.1 导入数据库

```bash
# 创建数据库
CREATE DATABASE expense_tracker;

# 导入表结构
mysql -u root -p expense_tracker < db/schema.sql
```

#### 2.2 修改配置文件

打开 src/main/resources/application.properties，修改数据库连接信息：
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/expense_tracker
spring.datasource.username=root
spring.datasource.password=你的密码
```



#### 2.3 启动后端服务

```bash
# 在项目根目录执行
./mvnw spring-boot:run
```
启动成功后显示：Started Application in xxx seconds




### 3. 前端部署步骤

#### 3.1 安装依赖

```bash
npm install
```


#### 3.2 配置后端地址

修改 .env 文件：
```text
VITE_API_BASE_URL = http://localhost:8080
```


#### 3.3 启动开发服务器
```bash
npm run dev -- --host
```

启动后会显示：

- Local: http://localhost:5173

- Network: http://192.168.x.x:5173

> 同 WiFi 下使用 Network 地址访问


#### 3.4 构建生产版本
```bash
npm run build
```

生成 dist 目录，可部署到静态服务器。





## 二、操作指南

### 1. 注册账号
1.打开系统首页，点击「注册」按钮

2.填写用户名、密码、邮箱

3.点击「注册」，提示成功后自动跳转登录页

> 用户名不能重复，密码建议使用 6 位以上字符


### 2. 登录系统

1.在登录页输入用户名和密码

2.点击「登录」，进入记账主页

> 登录成功后，Token 会自动保存，后续操作自动携带




### 3. 添加收支记录

#### 添加支出
1.点击「添加支出」按钮

2.填写金额（如：50）

3.选择分类（如：餐饮）

4.选择日期

5.可选填写备注

6.点击「保存」

#### 添加收入
步骤同上，类型选择「收入」

> 金额必须大于 0，日期默认当天


### 4. 查看收支列表
- 主页默认显示最近收支记录

- 可按月份筛选：选择月份 → 点击「查询」

- 可按分类筛选：选择分类 → 点击「查询」


### 5. 编辑/删除记录
#### 编辑
1.找到要修改的记录

2.点击「编辑」按钮

3.修改金额或分类

4.点击「保存」

#### 删除
1.点击「删除」按钮

2.确认删除

> 删除后不可恢复，请谨慎操作



### 6. 查看统计图表

1.点击顶部导航「统计」或「报表」

2.支出饼图：查看各分类支出占比

3.月度报表：查看总收入、总支出、结余



### 7. 预算设置
1.点击「预算」页面

2.选择分类（如：餐饮）

3.设置月度限额

4.点击「保存」

5.当支出超过限额时会收到提醒



### 8. 退出登录

点击右上角「退出」按钮，返回登录页




## 三、常见问题

### Q1：登录时提示“网络错误”
**原因**：后端服务未启动或前端配置的后端地址不正确。

**解决**：

1.确认后端已启动：./mvnw spring-boot:run

2.检查 .env 中的 VITE_API_BASE_URL 是否正确


### Q2：注册时提示“用户名已存在”
**原因**：该用户名已被注册。

**解决**：更换一个用户名重新注册。


### Q3：添加记录后刷新页面数据消失
**原因**：数据未成功写入数据库。

**解决**：

1.检查后端日志是否有报错

2.确认数据库连接正常


### Q4：饼图不显示
**原因**：当月无支出记录，或饼图接口未就绪。

**解决**：先添加一笔支出，再查看饼图。
